# Antigravity 手机端 Web IDE 监控面板设计文档

## 核心设计理念：基于上下文跟随与抽屉式布局

为了在有限的手机屏幕上实现接近完整 IDE 的掌控感，不应该将 PC 端的复杂面板生搬硬凑，而是采用 **"主界面对话流驱动 + 辅助界面随需滑动"** 的架构。

核心状态树仅需维护：
- `active_port`：当前用户正在查看的 Language Server 实例（一切数据的根）。
- `active_conv_id`：当前正在进行的会话 ID。
- `active_model`：当前选用的模型（默认由 LS 决定）。

## 视图 1：主控台（第一层 - 聊天与任务流）

主界面的职责是显示总览进度和接收用户指令。它 **强绑定** 顶部的 LS 实例选择器。

### 顶栏布局（两行精简版）

```text
┌──────────────────────────────────────────────┐
│ 🟢 test_reply ▼                 Gemini 3.1 Pro ▼ │  ← 第一行
│ ＋ 新对话                            会话 #1 ▼ │  ← 第二行
├──────────────────────────────────────────────┤
```

- **第一行**：左=项目选择器（点击弹出下拉列表切换工作区），右=模型选择器
- **第二行**：左=`+ 新对话`按钮（清空聊天、生成新conv_id），右=当前会话标签+下拉切换历史会话
- 用户名/套餐/额度信息移至设置页面（⚙️ 视图 7）

### 项目名称规则
workspace 字段为有损 URI 编码路径（下划线同时替代路径分隔符和存在于目录名中）。
后端 `api_server.py` 的 `_decode_workspace_name()` 使用 `os.path.exists()` 磁盘贪心匹配算法还原正确目录名，前端直接使用返回的 `display_name` 字段。

### 会话管理
- 每次发消息返回 `conv_id` 后，自动存入 `localStorage` 的会话列表（带时间戳+首条消息摘要）
- 点击「+ 新对话」→ 清空聊天区、置空 conv_id，下次发消息自动生成新会话
- 点击「会话 #N ▼」→ 弹出下拉列表选择历史会话，切换后设置 conv_id 并清空聊天区
- 后端 `GET /v1/instances/{port}/conversations` 目前为 TODO，暂不依赖

### 主聊天区

```text
│                                      │
│ 👤 你: 帮我写个请求天气的脚本           │
│                                      │
│ 🤖 AG: 好的，正在执行...                │  ← 任务进度节点
│   ▶ 查看详细执行过程 (5步)             │  ← 折叠的 Action Tree（默认收起）
│   ✅ 执行完毕，耗时 12.3 秒            │  ← 执行结论
│                                      │
│ 💬 输入你的指令...                [➤] │  ← 输入框发送 prompt
├──────────────────────────────────────┤
│ 📁   📷   >_   ⚙️                    │  ← 底部4按钮: 文件/截图/终端/设置
└──────────────────────────────────────┘
```

**顶栏数据来源**：
- 实例列表 → `GET /v1/instances`
- 用户 + 配额 → `GET /v1/ls/user-status`（提取 `userTier.name` 判断 Ultra/Pro）
- 模型选择器 → `GET /v1/ls/models`（label 做显示，model 做值）
- 切换模型 → 发消息时 `POST /v1/chat` 带 `model` 字段

**底部导航栏**：4 个按钮分别触发视图 2/6/5/7

```text
└──────────────────────────────────────┘
```
**数据来源**：`/v1/tasks/{id}` 的 `raw_steps` 中 `COMMAND_EXECUTION` 类型 step payload。

```text

│ (变暗的主界面)                        │

│ test_reply.txt                       │

│ Python 3.12.9                        │
│                                      │
│ 💬 输入命令...                  [➤]  │
```

**数据来源**：`POST /v1/system/exec`（command + cwd/port）。前端维护命令历史列表，每次执行追加输出。

## 视图 6：远程截图查看器（📷 按钮）

手机上一键查看电脑 IDE 桌面的实时截图。**最具冲击力的功能。**

```text
┌──────────────────────────────────────┐
│ ✕  桌面截图  [🔄 刷新]  [🖥️ / 🪟]   │  ← 全屏/窗口切换
├──────────────────────────────────────┤
│                                      │
│    ┌──────────────────────────┐      │
│    │ Antigravity IDE 窗口截图  │      │  ← 可双指缩放
│    │ (实际渲染 base64 JPEG)    │      │
│    │                          │      │
│    └──────────────────────────┘      │
│                                      │
│  1800×1200 152KB 2026-03-19 13:00    │  ← 元信息
└──────────────────────────────────────┘
```
**数据来源**：`GET /v1/system/screenshot?title=Antigravity&mode=window|full`。当 `mode=window` 时精确切除阴影；当 `mode=full` 时，自动探测并最大化所在屏幕，通过 API 获取屏幕分辨率再截取单块屏幕（防连屏暴走）。返回 base64 JPEG。

```text

  │
  ├── 视图 1 主控台 ──────→ GET /v1/tasks/{id}  (轮询)
```

## 前端架构演进策略

**当前**：`index.html` 单文件（~2000 行 / 85KB），外部依赖仅 `marked.js`（CDN），部署即用。

**拆分阈值**（3000+ 行或多人协作时）：

```
index.html          ← 骨架 HTML
css/style.css       ← 样式（含 Markdown 排版）
js/app.js           ← 状态管理 + 初始化
js/chat.js          ← 聊天收发 + Markdown 渲染
js/settings.js      ← 设置面板 + 系统监控
js/views.js         ← 文件树 / 终端 / 截图等视图
```

**模型适配**：`/v1/ls/models` 为纯透传（1 小时 TTL 缓存），前端 `detectVendor()` 按关键词自动分组，无需随模型升级维护。
