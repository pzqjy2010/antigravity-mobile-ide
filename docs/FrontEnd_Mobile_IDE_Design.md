# Antigravity Mobile IDE — 前端设计文档

## 核心设计理念

在有限的手机屏幕上实现接近完整 IDE 的掌控感，采用 **"主界面对话流驱动 + 辅助界面随需滑动"** 架构。

核心状态树（`frontend/js/state.js`）：
- `activePort`：当前 LS 实例端口
- `activeConvId`：当前会话 ID
- `activeModel`：当前选用模型

---

## 前端模块化架构

```
frontend/
├── index.html              ← HTML 骨架（纯结构，无内联 CSS/JS）
├── css/style.css           ← 全局样式（含从 JS 提取的语义化 class）
└── js/
    ├── app.js              ← 入口：导入所有模块 → window 暴露 → 初始化
    ├── state.js            ← 状态 + localStorage + SWR 缓存
    ├── ui.js               ← 抽屉/模态/消息追加
    ├── chat.js             ← 会话管理 + 消息收发 + AI 响应渲染
    ├── instances.js        ← 实例切换与状态恢复
    ├── models.js           ← 模型列表与选择
    ├── screenshot.js       ← 截图查看器 + 系统仪表盘
    ├── file-tree.js        ← 文件树 + 代码查看器
    ├── terminal.js         ← 远程终端
    ├── settings.js         ← 设置面板（探针/配额/进程/紧急停止）
    └── image-viewer.js     ← 全屏图片查看器
```

**设计原则**：
- ES Module 原生加载，无构建工具
- `app.js` 通过 `window.xxx = fn` 桥接 HTML `onclick`
- CSS 中使用语义化 class（如 `.window-card`、`.stat-value`），JS innerHTML 不含内联样式 → **大模型阅读友好**

## 多级缓存策略

| 层级 | 机制 | 说明 |
|------|------|------|
| **后端 - 进程** | 后台每 30s 自动刷新 | 请求永不阻塞，启动即预热 |
| **后端 - 会话** | 增量式缓存 | 首次全量，后续后台异步合并新增 |
| **后端 - 用户状态** | 30s TTL | 被动缓存 |
| **后端 - 模型** | 1h TTL | 极少变动 |
| **前端 - SWR** | `state.js` cacheGet/cacheSet | 10s fresh 窗口，先返回缓存再后台刷新 |
